Hiring manager Q&A (quick)

Short answers to common screening questions (expand for details).

How do you design CI/CD for a team?
  • Goal: small, repeatable steps with fast feedback and safe releases.
  • Approach: build/test/scan → artifact push → deploy via GitOps; keep secrets out of pipelines.
  • Tools: GitHub Actions / GitLab CI / Azure DevOps / Bitbucket; SonarQube where relevant.
What does “GitOps” mean in your day-to-day?
  • Desired state lives in Git (manifests/Helm values).
  • Argo CD applies and continuously reconciles; drift becomes visible and actionable.
  • Outcome: auditable changes, repeatable rollbacks, fewer “it works on my cluster” issues.
How do you manage secrets in Kubernetes?
  • Vault for storage/rotation; inject at runtime; avoid committing secrets to Git.
  • Access: least privilege, short-lived credentials where possible, and clear audit trails.
  • SSO: OIDC/Keycloak/Entra ID to simplify secure access to tooling.
How do you approach observability?
  • Metrics & alerts: Prometheus + Alertmanager, Grafana dashboards with actionable alerts.
  • Traces: OpenTelemetry when it helps find latency/cross-service issues.
  • Logs: centralized logging (e.g., Graylog) and uptime/synthetic checks (e.g., Uptime Kuma).
  • Incident analysis: New Relic where it adds value for infra + alerting workflows.
What’s your Terraform workflow?
  • Modules: reusable building blocks with clear inputs/outputs and consistent naming.
  • Automation: plan/apply through pipelines; policy checks as needed.
  • Governance: remote state, reviewable PRs, and drift prevention.
How do you scale workloads?
  • Baseline: right-size requests/limits and use HPA where it fits.
  • Event-driven: KEDA when scaling should track queues/events/metrics beyond CPU.
  • Outcome: better responsiveness and cost-efficiency.
How do you handle incidents?
  • Detection: actionable alerts (Prometheus/Alertmanager/New Relic), synthetic monitoring (Uptime Kuma).
  • Response: standardized incident channels (Slackbot), clear roles, and runbooks.
  • Post-mortem: blameless review, timeline reconstruction, and concrete action items to prevent recurrence.
What's your multi-cloud experience?
  • Providers: hands-on with AWS, GCP, Azure, DigitalOcean, and Huawei Cloud.
  • Migration: orchestrated full K8s cluster migration across cloud providers.
  • Strategy: Terraform abstractions to keep infra portable; avoid deep vendor lock-in where practical.
How do you reduce toil and manual work?
  • Automation: Bash/Python scripts for repetitive tasks, Slackbots for IAM and incident workflows.
  • Self-serve: built an internal IDP (OpsCentral) so teams can provision resources safely without SRE tickets.
  • Pipelines: centralized Helm chart pipelines, GitOps reconciliation to eliminate manual deploys.
